## Project 1 – Adowise

**What it is:** A technology education platform focused on practical web development learning.

**Why I built it:** Most developer education content is either too theoretical or too shallow. Adowise aims to give learners real, hands-on experience with modern web technologies.

**Tech stack:**
- Next.js (App Router)
- TypeScript
- PostgreSQL with Prisma ORM
- Vercel for deployment

**Key features:**
- User authentication with session management
- Structured course content with progress tracking
- Responsive dashboard for learners
- SEO-optimized public-facing pages

**What I learned building Adowise:**
Building a platform for learners forced me to think beyond code quality — I had to think about user experience, content organization, and how to make complex topics approachable. The SEO architecture was built intentionally, which is why it attracts organic traffic.

## Project 2 – JobGo

**What it is:** A job discovery platform designed to simplify the job search experience for candidates across India.

**Why I built it:** Job searching in India is fragmented and slow. JobGo is an attempt to make the experience faster and more focused.

**Tech stack:**
- Next.js with server-side rendering
- Node.js REST API backend
- MongoDB for flexible job listing storage
- Railway for backend deployment

**Key features:**
- Job listings with advanced filtering (role, location, type)
- Fast search with instant results
- Clean candidate-first UI
- Structured data markup for Google job search integration

**What this project demonstrates:**
JobGo shows my ability to handle data-heavy platforms with complex filtering logic — the same architecture used in e-commerce and real estate platforms. The schema markup and SEO foundation means job listings can appear directly in Google search results.

## Project 3 – Briefly.ai

**What it is:** An AI-powered content summarization and extraction tool.

**Why I built it:** Content overload is real. Briefly.ai helps users quickly extract key information from articles, documents, and long-form text using language model APIs.

**Tech stack:**
- Next.js frontend
- Integration with LLM APIs (server-side)
- Streaming responses for real-time output
- Vercel Edge Functions for low-latency performance

**Key features:**
- URL-based content summarization
- Document paste-and-summarize
- Streaming text output for responsive feel
- Clean, minimal UI focused on clarity

**What this project demonstrates:**
Building with AI APIs requires understanding both frontend UX patterns (streaming, loading states, error handling) and secure backend integration. I applied everything from React hook patterns to API rate limiting.

## Project 4 – WordFlow

**What it is:** A writing productivity application with AI-assisted features for writers and content creators.

**Why I built it:** Writing tools are either too simple or overwhelmingly feature-heavy. WordFlow targets the middle — a focused writing environment with just the right amount of assistance.

**Tech stack:**
- React with TypeScript
- Rich text editing with Slate.js
- Node.js backend for document management
- PostgreSQL for user data and document storage

**Key features:**
- Document creation and management
- AI writing suggestions integrated inline
- Clean editor interface without distractions
- Folder organization for documents

**What this project demonstrates:**
WordFlow shows my ability to build complex, interactive UIs with rich text editors — not just standard form-based interfaces. It also required thoughtful UX decisions about when and how to surface AI suggestions without disrupting the writing process.
